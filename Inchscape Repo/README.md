# Inchcape Suzuki collision-parts forecasting

This repository contains the data preparation and exploratory analysis for monthly Suzuki collision-part demand in Chile.  It is structured so that inputs, reproducible code, tables, and figures are kept separate.

## Repository layout

| Location | Purpose |
| --- | --- |
| `data/raw/` | Source sales extract; do not edit. |
| `data/external/` | Monthly external features supplied with the project. |
| `data/processed/` | Clean, model-ready collision-part data split by demand type. |
| `src/` | Reproducible preparation and analysis scripts. |
| `notebooks/` | Small, task-focused notebooks.  The two original notebooks are retained as archives. |
| `results/tables/` | Analysis outputs in CSV format. |
| `results/figures/` | Labelled PNG charts generated by the scripts. |
| `docs/` | Source data dictionary and project documentation. |

## Quick start

Create an environment with the packages in `requirements.txt`, then run:

```powershell
python src/prepare_data.py
python src/run_eda.py
```

The preparation step reads only the fields needed for analysis from the raw extract. It cleans the date and collision flag, classifies each collision SKU using the ADI/CV² method, and writes one CSV per demand type. The EDA step creates the CSV tables and PNG figures from those processed files.

## Models

The modelling code from the original notebooks is now organised rather than removed:

- `src/models/benchmarks.py`: naive, seasonal naive, 3- and 6-month moving averages, Croston, SBA, and TSB.
- `src/models/advanced.py`: XGBoost, LightGBM, Random Forest, and the two-stage LightGBM hurdle model for lumpy demand.

Run `python src/run_benchmarks.py` to reproduce baseline rolling-origin validation. It writes `benchmark_metrics.csv` and `benchmark_forecasts.csv` to `results/tables/`. The collision subset contains 28 months, so this runner uses an 18-month initial training window; the original notebook's 36-month window is still available in the function but cannot create a test fold with this data. The advanced models are optional; install `xgboost` and `lightgbm` before using them.

## Demand-type definitions

The classification uses average demand interval (ADI) and squared coefficient of variation (CV²), calculated from positive monthly demand.

| Demand type | Rule |
| --- | --- |
| Smooth | ADI ≤ 1.32 and CV² ≤ 0.49 |
| Intermittent | ADI > 1.32 and CV² ≤ 0.49 |
| Erratic | ADI ≤ 1.32 and CV² > 0.49 |
| Lumpy | ADI > 1.32 and CV² > 0.49 |

SKUs with no observed demand are kept separately as `collision_sales_no_demand.csv`, rather than being forced into one of the four analytic demand types.
